# Pre-1.0 Polish Design

Date: 2026-03-05

## Goal

Make the 1.0 release shockingly good. All items are non-breaking: documentation fixes, dead code removal, type hygiene, and test coverage gaps.

## Items

### 1. Fix incorrect type signatures in docs

Simple API tag fields are `string[]` (multi-value), but docs show `string`. Fix all affected pages: `docs/api/index.md`, `docs/guide/quick-start.md`, `README.md`, `LLMs.md`.

### 2. Remove `forceBufferMode` entirely

No consumers exist. Delete the property from all interfaces (`FolderScanOptions`, `LoaderOptions`, `SimpleLoaderOptions`, etc.), remove internal usage in `module-loader.ts`, `config.ts`, `deno-compile.ts`, and any test references.

### 3. Clean up `global.d.ts`

Replace 6 instances of `any` with `unknown` for Deno namespace stubs (`FsFile`, `SeekMode`, `OpenOptions`, `ChildProcess`, `CommandStatus`, and the `Deno` const). These shims prevent type errors in non-Deno environments.

### 4. Add mock-based unit tests for web-utils

6 untested functions: `setCoverArtFromCanvas`, `canvasToPicture`, `imageFileToPicture`, `displayPicture`, `createPictureDownloadURL`, `createPictureGallery`. Mock DOM/Canvas APIs and test logic: correct picture object construction, MIME types, error handling. Extend `tests/web-utils-unit.test.ts`.

### 5. Verify/add extended property roundtrip tests

Check whether `basic-tags.test.ts` covers albumArtist, composer, MusicBrainz IDs across formats. If not, add roundtrip assertions for MP3 and M4A.
